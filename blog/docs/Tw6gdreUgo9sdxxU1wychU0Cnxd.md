GPT 5.6 出来了。

官方宣传里说很强。但从我自己的使用体验来看，没有什么惊喜，反而明显感觉限制更多、选择更复杂、平台也更封闭了。 

## 1.1 选择更多了，但也更累了

首先是模型选择变复杂了。

以前我只需要在低、中、高之间选一个，大概知道自己是在省资源、平衡体验，还是追求效果。

现在又变成了太阳（sol）、地球（terra）、月亮（luna）这一套命名。官方可能觉得这是在给用户更多选择，但对我来说，它反而增加了心理负担。

我不想每次开始一个任务之前，还要先猜半天：这个模式到底适合什么场景？我该选哪个？会不会选错？

工具应该帮人减少决策，而不是把更多决策丢给用户。

## 1.2 模型更敏感了，使用体验也变差了

其次，模型明显更敏感了。

以前用 GPT 5.5，让它分析一款软件，或者讨论一些合规的数据获取思路，基本还能正常聊下去。但到了 GPT 5.6，感觉边界收得更紧了。

只要话题稍微碰到一点敏感区域，它就很容易直接拒绝。更麻烦的是，一旦这个对话里出现过敏感问题，后面的上下文好像也会被污染。你再问其它正常问题，它也可能继续拒答，甚至提示你去申请某种权限。

说实话，我不太赞同这种处理方式。安全当然重要，但如果安全研究、风控分析、数据获取这些能力只能由平台或大公司来做，普通开发者连讨论和学习的空间都没有，那这本身是不是也会形成一种新的不平等？

技术的安全边界，不应该最后变成少数人的特权。

## 1.3 Codex 也没那么开放了

如果想用 GPT 5.6，就必须用 GPT 账号登录。像我们这种使用 API 中转服务的人，只能继续用 GPT 5.5。

这件事让我很不舒服。你可以说这是商业策略，也可以说是生态控制，但对用户来说，感受就是平台边界越来越收紧。

以前我觉得 OpenAI 最大的吸引力之一，是它足够开放，开发者可以用自己的方式接入、组合、折腾。现在这种感觉正在变弱。

## 1.4 强不强我不知道，封闭我感受到了

OpenAI 宣传 GPT 5.6 有多强，我作为普通用户其实没有明显感受到。 

选择更复杂，限制更敏感，入口更封闭。

这才是我这次最直接的感受。

## 1.5 国内平台确实追上来了

最近我也明显感觉到，国内公司正在追上来。

和 Codex 对标的，有智谱的 ZCode、阿里的 Qoder、腾讯的 WorkBuddy。我都试用了一下，体验已经完全不输 Codex。

模型方面，GLM、Kimi、Qwen 在我日常开发工作里的表现也基本不差。

所以这件事也让我更确定：我们不应该只盯着一个平台。

感谢国内这些平台还在往前追。至少它们让普通开发者多了一些选择，也让我们不至于完全受制于人。
